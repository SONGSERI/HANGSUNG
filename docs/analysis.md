# 데이터 분석 컨텐츠

본 항목은 현재 정의된 데이터 구조(LOT, MACHINE_RUN 및 요약 테이블)를 기반으로  
수행 가능한 분석 범위를 정리한 것이다.  
실제 분석 과제는 데이터 분포 및 품질 검토 후 선정한다.

1. 정지 사유 Top (Pareto)
고객이 가장 보고 싶어 하는 화면
어떤 사유부터 줄여야 하는지 즉시 판단
KPI 감소 활동의 출발점
2. 설비 × 정지 사유 분포 (Heatmap)
특정 설비 고질 문제 vs 공통 문제 구분
“이 설비만 왜 자꾸 멈추냐” 질문에 바로 답 가능
3. 정지 사유별 점검 · 조치 가이드
AI 판단 ❌
행동 기준 제시 ⭕
고객이 요구한 “사유 해소를 위한 행위 조건”을 그대로 충족
